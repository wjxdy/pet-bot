# 🔄 PetBot 开发工作流

## 协作模式：全自动开发 + 用户确认

### 📋 工作流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  你提需求    │ ──▶ │  我开发功能  │ ──▶ │ 自动测试    │
└─────────────┘     └─────────────┘     └─────────────┘
                                              │
                                              ▼
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  功能上线    │ ◀── │  你确认 OK  │ ◀── │  测试通过   │
└─────────────┘     └─────────────┘     └─────────────┘
```

### 🎯 你的角色

1. **提出需求**：告诉我想要什么功能
2. **确认设计**：确认我的实现方案
3. **验收结果**：确认功能是否符合预期
4. **提供反馈**：告诉我需要调整的地方

### 🛠️ 我的角色

1. **设计功能**：设计实现方案
2. **编写代码**：完成开发
3. **自动测试**：运行测试脚本验证
4. **汇报结果**：向你报告测试结果
5. **修复问题**：根据反馈调整

### ✅ 测试流程

每次功能完成后，我会：

1. **编译测试**：`swift build`
2. **功能测试**：检查核心功能
3. **UI 测试**：模拟用户操作
4. **生成报告**：汇总测试结果

### 📱 通知方式

我会通过飞书告诉你：

```
🎉 新功能完成：聊天历史窗口

📊 测试结果：
✅ 编译通过
✅ 应用启动正常
✅ 窗口创建成功
✅ 菜单功能正常
✅ UI 交互正常

📝 使用说明：
点击菜单栏 PetBot → 打开聊天历史

请测试确认是否需要调整！
```

### 🔧 需要你的配合

1. **授予测试权限**（一次性）：
   - 系统设置 → 隐私与安全性 → 辅助功能
   - 添加 Terminal

2. **查看测试结果**：
   - 我会发送测试报告
   - 如有问题，我会附上日志

3. **确认或反馈**：
   - "功能 OK，继续下一个"
   - "需要调整 XXX"

### 🚀 示例对话

**你**：我想添加一个暗黑模式

**我**：收到！设计方案：

请确认方案，我开始开发。

---

**我**：✅ 暗黑模式开发完成！

📊 测试结果：
- 编译：通过
- 主题切换：正常
- 重启保持设置：正常

请测试确认。

---

**你**：可以，但默认应该是浅色模式

**我**：收到，修改默认值为浅色模式。

✅ 已修改并测试通过！
